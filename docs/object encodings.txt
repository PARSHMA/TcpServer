 Type and Encoding

Redis supports 7 object types. Each object type can have one or more internal encodings.

1. String
   Encodings:
   - raw
   - int
   - embstr (used for strings < 44 bytes)

2. List
   Encodings:
   - ziplist
   - linkedlist

3. Set
   Encodings:
   - intset
   - hashtable

4. Sorted Set (ZSet)
   Encodings:
   - ziplist
   - skiplist

5. Hash
   Encodings:
   - ziplist
   - hashtable

6. Module

7. Stream

Notes:
- Ziplist is memory-efficient and is used when the dataset is small.
- When the data grows beyond a configured threshold, Redis switches to a more scalable encoding (e.g., hashtable or skiplist depending on the data type).
- The object type visible to the client never changes (e.g., STRING, HASH, SET), only the internal encoding changes.
